# EduQuest AI

Рабочий репозиторий для подготовки к [Digital Future Hackathon](https://hackathon.lezo.io/).

Название репозитория пока условное. На этапе брейншторма мы не фиксируемся на конкретной идее, продукте, нише или формате. Главная цель - найти сильную AI-first проблему, которую реально довести до публичного MVP за один уикенд.

## Контекст

Хакатон предлагает четыре возможные ниши:

- Education;
- Utilities;
- Pet Tech;
- Aging & Longevity.

Education сейчас выглядит наиболее вероятным направлением из-за близости к образовательным продуктам, видеообучению и школьному или корпоративному обучению. Но это не финальный выбор: другие ниши тоже стоит рассматривать, если там найдется более сильная проблема.

- [`brainstorm/ideas.md`](brainstorm/ideas.md) - рабочий список продуктовых гипотез по разным нишам.
- [`brainstorm/edu-games-gpt5.5.md`](brainstorm/edu-games-gpt5.5.md) - отдельная ветка идей для обучающих игр по разным жанрам (GPT-5.5).
- [`brainstorm/edu-games-opus4.8.md`](brainstorm/edu-games-opus4.8.md) - отдельная ветка идей для обучающих игр по разным жанрам (Opus 4.8).

## Что важно для идеи

Каждую идею оцениваем по нескольким вопросам:

- есть ли реальная и понятная пользовательская боль;
- является ли AI основной ценностью продукта, а не декоративной фичей;
- можно ли собрать MVP, прототип или лендинг за один уикенд;
- можно ли объяснить идею в коротком питче;
- есть ли логика монетизации;
- можно ли проверить спрос маленькой Facebook-кампанией.

## Текущий этап

Сейчас проект находится на этапе брейншторма. Следующий шаг - собрать больше идей, сравнить их по критериям и выбрать короткий шортлист для глубокой проработки.
