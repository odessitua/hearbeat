# HearBeat — продуктовий аудит демо + специфікація правок

**Що це.** Повний аудит чинного `demo-mockup.html` очима судді, який відкриває лінк уперше, + специфікація правок з поясненнями UX/UI і того, ЩО ми показуємо. Наприкінці — рішення по адаптивному діалогу (безкоштовно/платно) і ML-промпти. До цього документа додається переписаний `hearbeat-demo.html`.

**Головний висновок одним рядком:** чинне демо — це якісний клікабельний макет-вітрина з трьома вкладками, а має бути продукт, у який суддя занурюється на один золотий шлях і переживає «вау». Різниця між «показуємо вкладки» і «суддя тестує ядро сам» — це різниця між середньою і виграшною оцінкою за критерієм Б (продукт/тех).

## Частина 1. Аудит — що я побачила, проходячи як суддя

### 1.1. Що працює (не чіпати)

| Сильне | Чому цінне |
|---|---|
| Вхідний дзвінок з великою кнопкою «Відповісти» | Точний золотий шлях, знайомий патерн, нульове тертя для 65+ |
| Три шари продукту присутні | Клієнт (дзвінок) → Рушій (аналіз) → Адмінка (сигнал сім'ї) — концептуально повно |
| Рушій із «записати свій бейзлайн → перевірка» | Найсильніша частина: доводить AI-core наживо, суддя тестує сам |
| Акустичні ознаки з дельтами (темп −24%, паузи +55%) | Конкретика, а не «емоційний бал 7/10» — видно baseline-логіку |
| Дисклеймер «не ставить діагнозів» | Захищає wellness-позиціонування (урок Kintsugi) |

### 1.2. Що слабке (за критеріями хакатону)

| Проблема | Критерій, по якому б'є | Чому це коштує балів |
|---|---|---|
| **Три вкладки замість одного шляху** | Б + весь пітч | Методичка: «одна сцена, що тримає весь фільм». Суддя сам обирає, куди клікати → розмита увага, немає веденого «вау» |
| **Стиль не збігається з брендом** | Цілісність | Демо в теплих зелено-персикових тонах (Nunito), пітч у sea-wave (Manrope, #0E6B8C) → виглядає як два різні продукти |
| **«Демо для журі», «ДЕМО», «Карта гілок» в UI** | Impact / вау | Видно риштування. Суддя має бачити HearBeat, а не його кухню |
| **Діалог — це вибір з кнопок, не розмова** | Б (AI-first) | Суддя не може «сказати що завгодно» → відчуття скрипту, не інтелекту. Це головна недокрученість value |
| **Золотий шлях не замикається** | Impact | Після дзвінка суддя не бачить, як САМЕ цей дзвінок оновлює сімейний сигнал. Магія «голос → сигнал» стається в іншій вкладці |
| **Емодзі в табах 👵👨‍👩‍👧⚙️** | UI-смак | Template-естетика, проти якої ти сама завжди виступаєш |
| **Немає явного «вау-моменту»** | Impact | Score не «стрибає» на очах — суддя бачить статику, а не оживлення |

### 1.3. Головна діагностика

Демо відповідає на питання «що вміє HearBeat?» (вітрина функцій), а має відповідати на «що я, суддя, щойно пережив?» (продукт). Ти сама сформулювала правильний напрям: у Рушії має бути **живий діалог, який суддя веде голосом/текстом**, отримує контекстні уточнюючі питання, а потім оцінка їде в Адмінку. Це перетворює демо з макета на продукт. Нижче — як це зробити, і чи можна безкоштовно.

## Частина 2. Специфікація правок

### 2.1. Архітектура: від «3 вкладки» до «1 золотий шлях + докази»

**Було:** рівнозначні вкладки Клієнт / Адмінка / Рушій — суддя блукає.

**Стало:** один вертикальний шлях, який веде суддю, а вкладки лишаються як «поглибити», не як «обери сам»:

```
ГОЛОВНИЙ ЕКРАН (те, що суддя бачить першим):
1. Вхідний дзвінок HearBeat  →  [Відповісти]
2. Жива розмова: HearBeat питає → суддя відповідає (голосом АБО текстом)
   → HearBeat ставить КОНТЕКСТНЕ уточнююче питання (тут AI-core)
3. Розмова завершується → на очах рахується acoustic index + summary
4. Одразу нижче: «Ось що побачить сім'я» → dashboard оживає,
   з'являється НОВИЙ рядок від щойно проведеного дзвінка
```

Рушій (технічні деталі акустики, JSON baseline, запис свого голосу) — лишається доступним нижче або в окремій секції «Під капотом / для технічних суддів», але НЕ як рівнозначна вкладка на вході.

**Чому так:** суддя проходить один замкнений цикл «голос → аналіз → сигнал сім'ї» БЕЗ вибору, куди клікати. Це і є «вау»: він сам щойно зробив дзвінок і побачив, як той перетворився на сигнал для доньки.

### 2.2. Живий адаптивний діалог — серце правки (те, що ти просила)

**Було:** суддя тисне кнопку з готовою реплікою Марії → скрипт. Відчуття «це запрограмовано».

**Стало:** суддя **говорить або пише що завгодно** → HearBeat розуміє зміст → ставить доречне уточнююче питання → веде розмову 3–4 репліки → завершує й аналізує.

Це технічно можливо безкоштовно (деталі в Частині 3). Саме тут «прибери AI — і продукт зникне»: без розуміння змісту немає контекстних питань.

**UX діалогу:**
- Велике поле/кнопка мікрофона: «Скажіть або напишіть, як пройшов день».
- Суддя говорить (Web Speech API розпізнає) АБО друкує (fallback).
- HearBeat відповідає контекстно: якщо суддя згадав «погано спав» → «через що не спалось — турбувало щось?»; якщо «все добре, гуляла» → «як приємно, а з ким?».
- Після 3–4 обмінів HearBeat дякує й завершує.
- Тоді рахується acoustic index (по темпу/паузах введеного голосу, якщо був мікрофон; або демо-значення для текстового шляху).

**Критично для сцени:** лишити **fallback на кнопки-репліки**, якщо мікрофон/LLM недоступні (Firefox, немає ключа, впав інтернет). Живий діалог — «wow», кнопки — «безпечний бекап». Обидва ведуть в одну оцінку.

### 2.3. Замкнути золотий шлях візуально

Після завершення розмови — НЕ перемикати вкладку. Одразу під телефоном:
1. Анімовано з'являється acoustic index (число «стрибає» від 100 до, скажімо, 78).
2. Статус фарбується (зелений «як зазвичай» / жовтий «варто подзвонити»).
3. Кнопка «Ось що побачить Олена →» плавно розкриває міні-дашборд, де **нагорі з'являється новий рядок — щойно проведений дзвінок**.

Це і є момент, коли суддя бачить: голос → сигнал сім'ї. Магія стається на очах, в одному потоці.

### 2.4. Ребрендинг у sea-wave

- Шрифт: Manrope (не Nunito).
- Палітра: `#0E6B8C` (основний), `#0A4F68` (глибокий), `#5BB8D4` (хвиля), `#F2733E` (акцент/увага), `#8892A0` (сірий), білі картки, `#F2F7F9` фон-панель.
- Прибрати теплі персиково-зелені (`--coral`, `--deep` зелений, `--paper` кремовий).
- Статуси: «як зазвичай» — спокійний синьо-зелений; «варто подзвонити» — помаранчевий `#F2733E` (не тривожно-червоний — це wellness, не алярм).

### 2.5. Прибрати риштування з UI

- Видалити «(демо для журі)», «ДЕМО», «Карта гілок діалогу», «Приклад розмови й того, як її аналізує AI-рушій».
- Емодзі в навігації → прибрати або замінити тонкими лінійними іконками.
- Технічні написи («acoustic_index», JSON baseline) лишити ТІЛЬКИ в секції «під капотом» для техно-суддів, не на головному шляху.

### 2.6. Підсилити value-меседжі (те, що продає)

На головному екрані, короткими підписами (не стіною тексту):
- Під дзвінком: «Виглядає як дзвінок від доньки. Батькам — нічого вчити.»
- Під аналізом: «Порівнюємо не з нормою для літніх, а з тим, як зазвичай звучить САМЕ ваша мама.»
- Під дашбордом: «Донька бачить один сигнал: як зазвичай — чи варто подзвонити сьогодні.»

Роадмап і монетизацію на демо НЕ виносити — це пітч (ти правильно зазначила). На демо максимум — маленький рядок «Сьогодні: ядро аналізу. Далі: адаптивна розмова, календар, кілька родичів» унизу, щоб показати, що це частина більшого.

## Частина 3. Адаптивний діалог — безкоштовно чи платно?

**Коротка відповідь: повністю безкоштовно, з двома шарами.**

### 3.1. Розпізнавання голосу — БЕЗКОШТОВНО (браузерний Web Speech API)

- `SpeechRecognition` / `webkitSpeechRecognition` вбудований у Chrome, Edge, Opera, Safari (14.1+ macOS, 14.5+ iOS). Підтримує українську (`lang="uk-UA"`). Безкоштовно, без ключа, без бекенду.
- **Застереження чесно:** аудіо в Chrome обробляється на серверах Google (потрібен інтернет); Firefox — вимкнено за замовчуванням. Тому ОБОВ'ЯЗКОВО fallback на текстове поле + кнопки, якщо API недоступний. Для демо на сцені це ідеально: показуєш голос у Chrome, а бекап-кнопки рятують, якщо щось.
- Синтез мови (щоб HearBeat «говорив») — теж безкоштовний браузерний `SpeechSynthesis`, українською.

### 3.2. Контекстні уточнюючі питання — БЕЗКОШТОВНО (free-tier LLM)

Щоб HearBeat розумів БУДЬ-ЯКУ відповідь і ставив доречне питання, потрібен LLM. Безкоштовні варіанти станом на 2026:
- **Google Gemini API (AI Studio)** — 1500 запитів/день на Gemini Flash, без картки, безстроково. Найкращий баланс якості й ліміту для української.
- **Groq** — Llama 3.3 70B, ~14 400 запитів/день, дуже швидко (важливо для «живого» відчуття), без картки, OpenAI-сумісний.
- Для демо на вікенд будь-якого з них із головою вистачить.

**Застереження:** free-tier означає, що промпти можуть використовуватись для тренування моделі — для демо з синтетичними даними це неважливо, але в реальному продукті з голосом літніх людей — тільки платний tier із no-training. Це чесний рядок для Q&A.

### 3.3. Гібридна архітектура демо (що я закладаю в HTML)

```
Шар 1 (завжди працює, 0 залежностей): кнопки-репліки + сценарний аналіз.
   ← безпечний бекап для сцени, працює офлайн, у будь-якому браузері.

Шар 2 (якщо є мікрофон): Web Speech API розпізнає мову судді.
   ← «вау»: суддя реально говорить.

Шар 3 (якщо вставлено безкоштовний API-ключ): LLM генерує контекстне
   уточнююче питання на будь-яку відповідь.
   ← «супер-вау»: HearBeat справді розуміє і веде розмову.
```

У переписаному HTML я закладаю **всі три шари** з плавною деградацією: без ключа працює на кнопках (як зараз, але краще); встав ключ у одну змінну — вмикається живий діалог. Так демо ніколи не зламається на сцені, але може показати повну силу.

## Частина 4. ML-промпти для контекстного діалогу

Якщо вмикаєте Шар 3 (LLM), ось промпти, які я закладаю в код. Вони підібрані так, щоб HearBeat звучав як тепла сім'я, а не робот/лікар — і збирав голосовий сигнал непомітно.

### 4.1. Системний промпт (роль HearBeat у розмові)

```
Ти — HearBeat, теплий голосовий чек-ін для літньої людини від імені її дорослої дитини.
Твоя мета: провести коротку (3–4 репліки) природну розмову, як телефонує турботлива родина.
Правила:
- Тон: теплий, простий, як донька/син, НЕ як лікар і НЕ як робот. Коротко.
- Питання короткі, по одному за раз. Реагуй на ЗМІСТ сказаного.
- Якщо людина згадала втому/біль/поганий сон — м'яко уточни ОДИН раз (через що, чи турбує щось).
- Якщо все добре — підтримай тему, спитай про приємне (хто заходив, що робила).
- НІКОЛИ не став діагнозів, не згадуй хвороби, не давай медичних порад.
- Після 3–4 обмінів тепло попрощайся і скажи, що передаси родині.
- Відповідай ЛИШЕ українською, однією короткою реплікою.
```

### 4.2. Промпт для summary (що піде в дашборд сім'ї)

```
На основі цієї розмови напиши для дорослої дитини ДВА рядки українською:
1. Коротко (1 речення), про що була розмова — по-людськи, без клінічних термінів.
2. Рекомендація дії: одне з — "Все як зазвичай" АБО "Можливо, варто подзвонити сьогодні".
Без діагнозів. Спирайся тільки на зміст розмови.
Формат JSON: {"summary": "...", "action": "..."}
```

### 4.3. Як рахується acoustic index (лишається на клієнті, без LLM)

Це вже є в демо і працює правильно — НЕ віддавати LLM. Акустика (темп, паузи, pitch, енергія) рахується з аудіо або береться зі сценарію, порівнюється з персональним baseline (індекс 100). LLM відповідає ТІЛЬКИ за зміст розмови (питання + summary), не за score. Це і є ваша сила: score — об'єктивна акустика проти baseline, не суб'єктивна оцінка LLM. Тримати цей поділ чітко — це прямий аргумент проти конкурентів на кшталт «емоційний бал від LLM».

## Підсумок: що змінює переписаний HTML

1. Один золотий шлях замість трьох вкладок (вкладки → «поглибити»).
2. Живий адаптивний діалог (голос + текст + LLM), з fallback на кнопки.
3. Замкнений цикл: розмова → score стрибає на очах → дашборд оживає з новим рядком.
4. Sea-wave брендинг (Manrope, #0E6B8C), прибране риштування.
5. Три шари деградації — ніколи не ламається на сцені, але показує повну силу з ключем.
6. Value-підписи, що продають, без стіни тексту.
